# Data Model: Insights Screen

**Feature**: 005-insights-screen
**Date**: 2026-02-10

## Existing Entities (No Changes)

### SleepEntry (from drift `sleep_entries` table)

The Insights screen reads from the existing `SleepEntry` data class generated by drift. No schema changes are required.

| Field | Type | Description |
|-------|------|-------------|
| id | String | UUID primary key |
| wakeDate | String | ISO date (YYYY-MM-DD), indexed |
| bedtimeTs | DateTime | Bedtime timestamp |
| wakeTs | DateTime | Wake timestamp, indexed |
| durationMinutes | int | Computed duration (1–1440) |
| quality | int | Quality rating (1–5) |
| note | String? | Optional note (max 280 chars) |
| createdAt | DateTime | Creation timestamp |
| updatedAt | DateTime | Last update timestamp |

**Indexes**: `idx_sleep_entries_wake_date`, `idx_sleep_entries_wake_ts`

## New Computed Types (Dart only, not persisted)

These types exist only in memory as computation results. They are not stored in the database.

### DurationDataPoint (reuse existing typedef)

Already defined in `lib/providers/home_providers.dart`:

```
typedef DurationDataPoint = ({String date, int durationMinutes});
```

For Insights, `durationMinutes` is the **sum** of all entries for that day (differs from Home which uses the latest entry).

### QualityDataPoint (new typedef)

```
typedef QualityDataPoint = ({String date, double averageQuality});
```

| Field | Type | Description |
|-------|------|-------------|
| date | String | ISO date (YYYY-MM-DD) |
| averageQuality | double | Mean quality rating for all entries on this day, rounded to 1 decimal |

### PatternSummary (new class)

```
class PatternSummary {
  final int avgDuration7d;       // Average duration in minutes (last 7 days)
  final int avgDuration30d;      // Average duration in minutes (last 30 days)
  final double avgQuality30d;    // Average quality (last 30 days), 1 decimal
  final String consistencyText;  // Plain-language bedtime consistency
  final String? bestDay;         // Day-of-week name (e.g., "Saturday"), null if no data
  final String? worstDay;        // Day-of-week name (e.g., "Wednesday"), null if no data
}
```

| Field | Type | Description |
|-------|------|-------------|
| avgDuration7d | int | Mean duration in minutes across entries with wakeDate in last 7 days |
| avgDuration30d | int | Mean duration in minutes across entries with wakeDate in last 30 days |
| avgQuality30d | double | Mean quality rating across entries with wakeDate in last 30 days |
| consistencyText | String | Human-readable bedtime consistency (see Computation Rules below) |
| bestDay | String? | Weekday name with highest avg duration in 30d window; null if no data |
| worstDay | String? | Weekday name with lowest avg duration in 30d window; null if no data |

## Computation Rules

### Duration Aggregation (7-day bar chart)

1. Query entries where `wakeDate` is within `[today - 6 days, today]`
2. Group by `wakeDate`
3. Per day: **sum** all `durationMinutes` values
4. Fill missing days with 0

### Quality Aggregation (30-day line chart)

1. Query entries where `wakeDate` is within `[today - 29 days, today]`
2. Group by `wakeDate`
3. Per day: compute **mean** of `quality` values, round to 1 decimal
4. Only include days that have entries (no zero-fill)

### Average Duration (7d / 30d)

1. Filter entries by the respective date window
2. Sum all `durationMinutes`, divide by count of entries
3. Round to nearest integer (minutes)

### Average Quality (30d)

1. Filter entries to 30-day window
2. Sum all `quality` values, divide by count
3. Round to 1 decimal place

### Bedtime Consistency (30d)

1. Filter entries to 30-day window
2. Convert each `bedtimeTs` to minutes-since-midnight:
   - `minutes = hour * 60 + minute`
   - If `hour < 12` (morning bedtime — unusual late night), add 1440 to normalise
3. Compute standard deviation of the normalised minutes
4. Map to plain-language text:
   - < 2 entries: "Not enough data for consistency"
   - stddev < 15: "Your bedtime is very consistent"
   - stddev 15–30: "Your bedtime is fairly consistent (about X minutes variation)"
   - stddev 30–60: "Your bedtime varies by about X minutes"
   - stddev 60+: "Your bedtime varies widely (over X hours)"
   - Where X = stddev rounded to nearest 5 minutes (for 30–60 range) or nearest hour (for 60+ range)

### Best/Worst Day of Week (30d)

1. Filter entries to 30-day window
2. Group by `DateTime.weekday` of `wakeTs` (1=Monday...7=Sunday)
3. Per weekday: compute mean `durationMinutes`
4. Best day = weekday with highest mean duration
5. Worst day = weekday with lowest mean duration
6. Display as full weekday name via `DateFormat.EEEE()`
7. If only 1 weekday has data, best and worst are the same day

## Data Flow

```
AppDatabase.listEntries(fromDate, toDate)
  → List<SleepEntry>
    → insightsDataProvider (FutureProvider)
      → InsightsCalculator.computeDurationChart(entries, today)
        → List<DurationDataPoint> (7 items)
      → InsightsCalculator.computeQualityChart(entries, today)
        → List<QualityDataPoint> (0–30 items, only days with data)
      → InsightsCalculator.computePatternSummary(entries, today)
        → PatternSummary
```
