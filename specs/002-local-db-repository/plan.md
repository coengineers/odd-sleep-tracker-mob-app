# Implementation Plan: Local Database & Repository Layer

**Branch**: `002-local-db-repository` | **Date**: 2026-02-09 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/002-local-db-repository/spec.md`

## Summary

Implement the local persistence layer for SleepLog using drift (SQLite ORM) with a repository pattern. This delivers full CRUD operations for sleep entries with input validation, computed fields (duration, wake date), and Riverpod integration — all on-device with zero network calls. Unit tests use in-memory SQLite for isolation.

## Technical Context

**Language/Version**: Dart 3.10+ / Flutter 3.38+ (stable channel)
**Primary Dependencies**: drift ^2.31.0, drift_flutter ^0.2.0, uuid ^4.0.0, path_provider ^2.0.0 (runtime); drift_dev ^2.31.0, build_runner ^2.4.0 (dev)
**Storage**: SQLite via drift (on-device only)
**Testing**: flutter_test + drift NativeDatabase.memory() for in-memory SQLite
**Target Platform**: iOS and Android
**Project Type**: Mobile (Flutter)
**Performance Goals**: List 365 entries in ≤500ms (excluding cold start)
**Constraints**: Zero network calls, offline-only, on-device storage
**Scale/Scope**: Single table (sleep_entries), ~365 rows/year typical usage

**Structure Decision**: Follows the existing D0 pattern of flat feature directories under `lib/`. The `database/` directory contains drift-specific code (tables + generated files). The `models/` directory holds domain input/output types. The repository is implemented directly in the database class as drift DAOs (no separate `repositories/` directory) — this avoids an unnecessary abstraction layer since drift IS the repository implementation. Riverpod providers in `providers/` expose the database and repository to the widget layer.

## Complexity Tracking

> No violations — all gates pass without justification needed.

| Decision | Rationale |
|----------|-----------|
| Repository methods on AppDatabase (no separate Repository class) | drift DAOs already provide type-safe CRUD. A separate interface/impl adds indirection with no benefit — there's only one storage backend (local SQLite). Validation lives in a helper function consumed by the database methods. |
| store_date_time_values_as_text: true | PRD specifies ISO datetime TEXT columns. Drift's text mode stores DateTimes as ISO 8601 strings, matching the PRD schema exactly. |
| UUID v4 (not v7) | PRD specifies UUID for IDs. v4 is the standard choice; v7 (time-ordered) adds no benefit since we order by wake_ts, not by ID. |
